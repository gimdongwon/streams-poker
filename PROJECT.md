# 🎮 STREAMS POKER (스트림즈 포커) - 프로젝트 정리

## 1. 게임 개요

> **장르**: 전략 카드 게임 / 멀티플레이 / 라운드 기반
>
> 10라운드 동안 매 라운드마다 공개되는 카드 1장을 10개 슬롯 중 하나에 배치하고,
> 최종적으로 완성된 10장의 카드 조합으로 포커 점수를 계산하여 승부를 결정하는 게임.

---

## 2. 핵심 게임 규칙 요약

### 2.1 덱 구성

- **54장**: 52장 (A~K × 4문양: ♠스페이드, ♥하트, ♦다이아몬드, ♣클로버) + **조커 2장**
- 매 게임 54장에서 **랜덤 10장**을 뽑아 사용
- 모든 플레이어가 **동일한 10장의 카드**를 받음 → 전략(배치 위치)으로 승부

### 2.2 라운드 진행

- 총 **10라운드**, 매 라운드 카드 **1장 공개**
- 플레이어는 **10초 내**에 10개 슬롯 중 하나에 카드 배치
- 시간 초과 시 → **가장 왼쪽 빈 슬롯에 자동 배치**
- **한 번 배치한 카드는 재배치 불가** (MVP 기준)
- 10라운드 종료 후 10장으로 조합 분석 & 점수 계산
- 최대 **10명** 동시 플레이

### 2.3 조합 적용 규칙

- 한 카드 세트는 **오직 하나의 조합**에만 사용 가능 (복수 조합 불가)
- 스트레이트 계열은 **상위 1개만** 적용
- 스트레이트 플러시는 스트레이트/플러시와 **중복 불가**
- 숫자 기반 조합(페어/트리플 등)은 카드가 겹치지 않는 한 **여러 개 적용 가능**
- **슬롯 배치 순서가 스트레이트 판정에 영향** → 연속된 슬롯의 카드로 스트레이트 구성
- 모든 조합 점수를 **합산**해 총점 결정

### 2.4 조커 카드 규칙

- 54장 중 **2장이 조커**
- 조커는 **와일드카드**: 어떤 카드든 대체 가능
- 조커는 **하나의 조합에만** 사용됨
- **가장 높은 점수**를 만들 수 있는 조합으로 자동 계산 (베스트 케이스)
- 조커 공개 시 **특별 애니메이션** (펑 터지는 효과)

### 2.5 승리 조건

- 총점이 가장 높은 플레이어 승리
- 동점 시: 최고 등급 조합 비교 → 다음 조합 비교 → 하이카드 비교

## 4. 기술 스택

### Frontend

| 기술 | 용도 | 선정 이유 |
| --------------- | ---------------------- | -------------------------------------------------- |
| **Next.js 15** | 프레임워크 (App Router) | SSR/SSG, API Routes, 라우팅 내장 |
| **TypeScript** | 타입 안전성 | 카드/게임 로직의 복잡한 타입 관리에 필수 |
| **TailwindCSS** | 스타일링 | 빠른 UI 개발, 반응형 대응 용이 |
| **Zustand** | 클라이언트 상태관리 | 가볍고 직관적, 게임 상태 관리에 적합 |
| **Framer Motion** | 애니메이션 | 카드 배치/이동 애니메이션에 활용 |

### Backend / 실시간 통신

| 기술 | 용도 | 선정 이유 |
| ----------------------- | -------------------- | ------------------------------------------- |
| **Next.js API Routes** | REST API | 방 생성/관리, 게임 초기화 등 |
| **Socket.io** | 실시간 멀티플레이 | 라운드 진행, 타이머 동기화, 카드 배치 동기화 |

### 인프라 (MVP 이후)

| 기술 | 용도 | 선정 이유 |
| ------------ | -------------- | ------------------------------ |
| **Vercel** | 배포 | Next.js 최적 배포 환경 |
| **Redis** | 세션/방 관리 | 빠른 인메모리 데이터 처리 |
| **Supabase** | DB/인증 | 유저 관리, 전적 기록 저장 |

## 9. 라우팅 & 멀티플레이 시스템 (v2)

### 9.1 페이지 구조

| 경로 | 설명 |
| --- | --- |
| `/` | → `/lobby` 리다이렉트 |
| `/lobby` | 로비 페이지 (싱글 모드 / 멀티플레이 모드 선택) |
| `/room/[roomId]` | 멀티플레이 대기방 (플레이어 목록, Ready/Start) |

### 9.2 플로우

```
[로비 /lobby]
  ├── 싱글 모드 → /game/solo (즉시 게임 시작)
  └── 멀티플레이
       ├── 방 만들기 → /room/[생성된roomId] (방장으로 입장)
       └── 방 참여하기 → 코드 입력 → /room/[roomId] (참가자로 입장)

[대기방 /room/[roomId]]
  ├── 플레이어 목록 표시 (닉네임, Ready 상태)
  ├── 각 플레이어가 Ready 버튼 클릭
  ├── 모든 플레이어 Ready 상태 → 방장만 Start 버튼 활성화
  └── 방장이 Start → 모든 플레이어 /game/[roomId]로 이동

[게임 /game/[roomId]]
  ├── 기존 게임 로직 동일
  ├── 모든 플레이어 동일 카드 수신
  └── 게임 종료 → 결과 비교 → 다시하기 or 로비로
```

### 9.3 대기방 상세 기능

- **방 생성**: 6자리 랜덤 코드 생성 (예: `A3K9F2`)
- **방 참여**: 코드 입력으로 참여
- **최대 인원**: 10명
- **플레이어 상태**: `waiting` → `ready`
- **방장 권한**: 게임 시작 (Start) 버튼은 방장만 가능
- **시작 조건**: 최소 2명 + 모든 플레이어 Ready
- **닉네임**: 입장 시 닉네임 입력 (로비에서)
