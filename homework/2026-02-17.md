## 핵심 로직 상세 (내일 작업에 필요한 컨텍스트)

### 포커 판정 엔진 (`evaluator.ts`)

평가 순서:
1. **스트레이트 계열 먼저** (연속 슬롯 기반, 위치 의존) — 스트레이트 플러시 > 일반 스트레이트, 상위 1개만 적용
2. **플러시 vs 숫자 조합 최적화** — 두 시나리오 비교:
   - 옵션 A: 플러시 먼저 → 남은 카드로 숫자 조합
   - 옵션 B: 숫자 조합 먼저 → 남은 카드로 플러시
   - 더 높은 총점 선택

조커 처리:
- 각 조합 검사 시 조커를 와일드카드로 취급
- 하나의 조합에만 사용 (used set으로 관리)
- `card.id`로 중복 방지

### 게임 스토어 (`gameStore.ts`)

상태 머신: `idle` → `playing` → `round_end` → `playing` → ... → `game_over`
- `startGame()`: 덱 셔플, 첫 카드 공개, 타이머 시작
- `placeCard(slotIndex)`: 카드 배치 → 500ms 후 `nextRound()`
- `autoPlaceCard()`: 빈 슬롯 중 **랜덤** 선택하여 자동 배치
- `tickTimer()`: 1초씩 감소, 0 도달 시 `autoPlaceCard()`

### 방 스토어 (`roomStore.ts`)

- `createRoom()`: 6자리 코드 생성, 방장으로 입장
- `joinRoom(code)`: 코드로 참여
- `toggleReady()`: Ready 토글
- `startGame()`: 방장 전용, 모든 플레이어 Ready + 2명 이상 조건
- `generateResults(myScore, myCombinationNames)`: 게임 종료 시 다른 플레이어 점수 시뮬레이션 (현재 로컬 랜덤)

## 확정된 게임 규칙

- 54장 덱 (52 + 조커2), 매 게임 랜덤 10장
- 모든 플레이어 동일 카드
- 10라운드, 10초 타이머, 시간 초과 시 **랜덤 빈 슬롯**에 배치
- 재배치 불가
- 조커: 와일드카드, 1개 조합에만, 베스트 케이스 자동
- 스트레이트: 연속 슬롯 필수
- 플러시: 위치 무관
- 한 카드 = 하나의 조합에만 사용
- 최대 10명
