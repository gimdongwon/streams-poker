# TENTENS - 작업 일지 (2026-02-18)

## 오늘 완료한 작업

### 1. 멀티플레이 라운드 동기화 시스템 구현
> 기존: 각 플레이어가 독립적으로 타이머를 돌려 서로 다른 시간에 게임 종료
> 변경: 서버가 라운드를 동기화 — 모든 플레이어가 배치 완료해야 다음 라운드 진행

**서버 (`server.ts`)**
- `Room` 타입에 `currentRound`, `roundPlacements` (Set), `roundTimer` 추가
- `startRoundTimer` / `advanceRound` / `checkRoundCompletion` 헬퍼 함수 추가
- `game:placed` 이벤트 핸들러: 플레이어 배치 추적 + 전체 배치 현황 브로드캐스트
- 모든 플레이어 배치 완료 → 800ms 후 `game:nextRound` emit
- 15초 안전 타이머: 클라이언트 미응답 시 자동 진행
- 플레이어 이탈 시 라운드 완료 체크 추가

**gameStore**
- `placeCard`: `setTimeout → nextRound()` 자동 진행 제거, `round_end` 상태만 설정
- `nextRound`: phase 안전 체크 추가 (`phase !== "round_end"` 시 무시)

**roomStore**
- `roundPlacedPlayers` 상태 추가 (각 라운드 배치 현황)
- `emitPlaced(round)`: 서버에 배치 완료 알림
- `game:playerPlaced`, `game:nextRound` 소켓 리스너 추가
- `game:nextRound` 수신 시 직접 `useGameStore.getState().nextRound()` 호출

**GameScreen**
- 싱글: 기존처럼 500ms 후 자동 진행
- 멀티: 배치 후 서버에 `game:placed` emit → 대기 UI 표시
- 대기 UI: 카드 뒷면(flip 애니메이션) + 플레이어별 배치 상태(✅/⏳) + 카운터

### 2. 결과 페이지 UI 개선
- 가로 분할(`landscape:flex-row`) → **세로 풀 스크롤**로 레이아웃 변경
- 결과/점수가 전체 너비를 사용하여 넓게 표시, 보드는 하단 스크롤
- ScoreBoard 텍스트 크기 전반적 증가 (조합명 `text-sm`, 카드정보 `text-xs`, 점수 `text-sm`, 총점 `text-2xl`)

### 3. `solo` → `single` 네이밍 변경
- 라우트: `/game/solo` → `/game/single`
- 타입: `"solo" | "multi"` → `"single" | "multi"`
- 변수: `isSolo` → `isSingle`
- 관련 파일 전체 일괄 수정 (page.tsx, GameScreen.tsx, lobby/page.tsx, roomStore.ts, 문서)

### 4. 조합 판정 규칙 변경 — 인접 슬롯 필수
**숫자 조합 (페어/트리플/포카드)**
- 기존: 슬롯 위치 무관하게 같은 숫자면 조합 성립
- 변경: **반드시 인접 슬롯**에 있어야 조합 성립
- 슬라이딩 윈도우 방식으로 `findNumberCombinations` 전면 재작성
- 포카드: 연속 4칸 / 트리플: 연속 3칸 / 페어: 연속 2칸
- 투페어/풀하우스: 각 서브 조합이 개별적으로 인접이면 OK (서로 간은 떨어져도 됨)

**플러시**
- 기존: 위치 무관, 같은 문양 5장
- 변경: **인접 5칸**에서 같은 문양이어야 성립
- `findConsecutiveGroups(slots, 5)` 기반으로 재작성

### 5. 동점 타이브레이커 로직
- `calculateTiebreaker()` 함수 추가: 조합 카드의 숫자 합계 (A=14, K=13 ... 2=2)
- 클라이언트: 게임 종료 시 `tiebreaker` 값 서버에 전송
- 서버: 1차 총점 비교 → 동점 시 2차 타이브레이커(카드 숫자 합) 비교로 순위 결정

### 6. 게임 종료 시 상태 초기화
- "로비로 돌아가기" 클릭 시 `resetGame()` + `resetRoom()` 모두 호출
- 새 방 입장 시 이전 게임 결과 페이지로 빠지는 버그 해결

## 트러블슈팅

### 🔥 라운드 동기화 후 다음 카드가 안 나오는 문제
**증상**: 라운드 1은 정상 동기화되지만, 라운드 2부터 `game:placed`가 전송되지 않고 safety timeout(15초)으로만 진행

**원인**: `roundSyncReady` useEffect에서 `consumeRoundSync()`를 호출하면 `roundSyncReady`가 false로 변경 → React useEffect cleanup 실행 → `setTimeout(() => nextRound(), 300)` 타이머가 취소됨

**해결**: `roundSyncReady` 플래그 방식을 완전히 제거. roomStore의 `game:nextRound` 소켓 리스너에서 직접 `useGameStore.getState().nextRound()`를 호출하여 React useEffect dependency 문제를 회피

### 🔥 게임 종료 후 새 방 입장 시 결과 페이지 노출
**증상**: 게임이 끝나고 로비로 돌아간 뒤 새 방에 입장하면 이전 게임의 결과 화면이 표시됨

**원인**: `handleBackToLobby`에서 `resetRoom()`만 호출하고 `resetGame()`은 호출하지 않아 gameStore의 `phase: "game_over"` 상태가 유지됨

**해결**: `handleBackToLobby`에 `resetGame()` 호출 추가

## 확정된 게임 규칙 (업데이트)

- 54장 덱 (52 + 조커2), 매 게임 랜덤 10장
- 모든 플레이어 동일 카드
- 10라운드, 10초 타이머, 시간 초과 시 랜덤 빈 슬롯에 배치
- 재배치 불가
- 조커: 와일드카드, 1개 조합에만, 베스트 케이스 자동
- **모든 조합은 인접 슬롯 필수**:
  - 스트레이트: 연속 5칸
  - 플러시: 연속 5칸, 같은 문양
  - 포카드: 연속 4칸, 같은 숫자
  - 트리플: 연속 3칸, 같은 숫자
  - 페어: 연속 2칸, 같은 숫자
  - 풀하우스/투페어: 각 서브 조합이 개별 인접이면 OK
- 한 카드 = 하나의 조합에만 사용
- **멀티플레이 라운드 동기화**: 모든 플레이어 배치 완료 시 다음 라운드 진행
- **동점 시 타이브레이커**: 조합 카드의 숫자 합(A=14)이 높은 쪽이 승리
- 최대 10명
